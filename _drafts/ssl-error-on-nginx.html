seen on Docker registry:

tls: oversized record received with length 28012

ssl_error_rx_record_too_long on nginx

Today I encountered a problem with nginx that was caused by "long" ssl keys. Our old keys had a length of just 1024 bits, but the new ones had 2048 which introduced a problem. In firefox the error "ssl_error_rx_record_too_long" showed up. In chrome "Error 107 (net::ERR_SSL_PROTOCOL_ERROR)" appeared.

The solution is really simple, if you know how to do it.
Open the vhost configuration of your site and edit this line:
listen 443;
 
change it into
listen 443 ssl;
 
Now you're done!
